# Промт для генерации презентации защиты ВКР

> Промт самодостаточный — его можно использовать в новом чате с Claude или другой моделью без знания предыдущей истории.

---

## ПРОМТ (копировать целиком)

```
Создай презентацию защиты ВКР в формате .pptx и текст доклада в .md.

КОНТЕКСТ ПРОЕКТА
================
- Название работы: «Разработка кроссплатформенного приложения для отслеживания силовых тренировок с локальной ML-моделью прогнозирования одноповторного максимума (1ПМ) и облачной синхронизацией».
- Рабочее название продукта: FitApp.
- ВУЗ: МТУСИ (Московский технический университет связи и информатики).
- Язык: русский.
- Стек: .NET MAUI 9 (клиент, Android + Windows), ASP.NET Core 9 (сервер), PostgreSQL/Neon, LightGBM → ONNX Runtime для ML.
- Ключевые особенности: offline-first, локальный ML без подписки, планировщик нагрузки, синхронизация Last-Write-Wins, open source.

ТРЕБОВАНИЯ К ПРЕЗЕНТАЦИИ
========================
Формат: .pptx, 12 слайдов, layout 16:9.
Длительность доклада: 7–8 минут.
Цветовая палитра — академическая (тёмно-синий 1E2761 как основной, ледяной 
CADCFC как светлый, коралловый F96167 как акцент для ключевых цифр).
Шрифт: Calibri (читается с проектора).

СТРУКТУРА СЛАЙДОВ
=================
1. Титульный — тёмный фон, тема, ВУЗ, автор, руководитель, год.
2. Актуальность — 4 буллета слева + карточка «Ниша FitApp» справа.
3. Цель и задачи — выделенный блок с целью + нумерованный список из 6 задач.
4. Сравнительный анализ — таблица: 5 продуктов (Strong, Hevy, FitNotes, Jefit, FitApp) × 7 критериев (Windows, Offline, RPE, ML-прогноз 1ПМ, Планировщик, Синхронизация, Монетизация). Колонка FitApp выделена коралловым.
5. Архитектура — три блока: Клиент (.NET MAUI 9, MVVM + 4 сервиса + хранилище), Сервер (ASP.NET Core 9, 3 эндпоинта), БД (PostgreSQL/Neon).
6. Стек технологий — 4 карточки в ряд: Клиент / Сервер / БД / ML-инфраструктура.
7. ML-модуль (датасет) — три большие плашки слева (763 322 подходов, 500 спортсменов, 118 350 пар) + 4 группы признаков справа (29 признаков всего).
8. ML-модуль (результаты) — гистограмма сравнения 3 моделей по MAE (Naive 2.359, LinReg 2.154, LightGBM 1.437) + большая плашка «−39%» + сравнение ONNX (медиана 1,6×10⁻⁵ кг, p99 0,082 кг).
9. Облачная синхронизация — диаграмма последовательности с 3 лайфлайнами (Клиент, Сервер, PostgreSQL) и 4 шагами + поясняющая строка про LWW.
10. Тестирование — функциональное (24 теста, 8 ключевых сценариев), интеграционное (4 сценария), таблица производительности (Windows/Android), плашка UX-теста с 6 добровольцами.
11. Выводы — широкая плашка «Цель достигнута» + 3 столбца: Реализовано / Научная значимость / Практическая ценность.
12. Спасибо за внимание — тёмный фон, центрированный текст «Готов к вопросам».

ОФОРМЛЕНИЕ КАЖДОГО СЛАЙДА (НЕ ТИТУЛЬНОГО)
==========================================
- Верхняя тонкая полоса основным цветом.
- Заголовок слайда: 24pt bold, тёмно-синий.
- Подвал: «FitApp — ВКР, МТУСИ, 2026» слева + номер слайда справа, 10pt MUTED-цвет.
- Никаких «акцентных линий под заголовком» (признак AI-сгенерированных слайдов).

ИНСТРУМЕНТЫ
============
Использовать pptxgenjs (Node.js). Установка:
    npm install -g pptxgenjs
Генерация:
    node build_presentation.js
Запретные приёмы из pptxgenjs:
  • НЕ использовать «#» в hex-цветах (только "1E2761", не "#1E2761").
  • НЕ кодировать прозрачность в hex (использовать свойство opacity).
  • НЕ использовать unicode-буллеты «•» (только bullet: true).
  • НЕ переиспользовать объект options между addShape/addText вызовами (мутируется).

ТЕКСТ ДОКЛАДА
=============
Отдельный .md-файл, в котором для каждого слайда — отрывок речи на 30–60 секунд.
Темп — 120 слов/мин, всего ~960 слов.
Стиль — академический, без воды, с акцентом на цифры.
В конце — раздел «Возможные вопросы комиссии и ответы» (6 вопросов с ответами).

ДОПОЛНИТЕЛЬНО
=============
- Файлы положить в vkr/presentation/.
- После генерации .pptx запустить markitdown для QA: проверить, что все слайды содержат ожидаемый текст и таблицы.
- Если есть soffice + pdftoppm — конвертировать в JPG и визуально проверить через subagent.
- Не коммитить автоматически; в конце ответа предложить название коммита.
```

## Что выдаёт промт на выходе

| Файл | Назначение |
|---|---|
| `vkr/presentation/build_presentation.js` | Генератор .pptx (Node.js) |
| `vkr/presentation/presentation_zaschita.pptx` | Готовая презентация, 12 слайдов |
| `vkr/presentation/tekst_zaschity.md` | Текст защиты с разбивкой по слайдам + ответы на вопросы |

## Как переиспользовать

1. Скопировать раздел «ПРОМТ» в новый чат.
2. Если изменились данные ВКР (другие цифры, новые слайды) — отредактировать соответствующий блок до запуска.
3. Запросить «выполни этот промт».
