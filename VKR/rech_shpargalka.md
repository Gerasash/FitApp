# Шпаргалка для защиты — FitApp

Маркеры-опоры: по ним говоришь своими словами. Полный текст — `rech_kratko.md`. Цифры держать в голове: **30 признаков · MAPE 2,5% · MAE 1,8 кг · 500 атлетов · 78 тыс. пар · горизонт 28 дней**.

---

**1. Титул**
→ Тема. Кроссплатформенное приложение + прогноз силовых показателей.

**2. Актуальность / проблема**
→ Учёт нужен, теряется «на глаз».
→ У аналогов нет прогноза. **Проблема:** нет офлайн-приложения с прогнозом.

**3. Цель и задачи**
→ Цель — эффективность тренировок через автоматизацию + рекомендации.
→ 4 задачи: анализ → приложение (.NET MAUI) → модель → тестирование.

**4. Архитектура**
→ MVVM, стандарт MAUI: интерфейс отдельно от логики.
→ 3 слоя: представление / модели представления / данные (SQLite).
→ DI + MAUI Shell → единый код Android и Windows.

**5. Интерфейс**
→ Слева ПК, справа телефон — единый код, экраны идентичны.
→ «Главная» — сводка + график. «Тренировки» — учёт подходов (вес/повторы/RPE).

**6. ML — как устроен**
→ Прогноз 1ПМ на 28 дней. Два этапа.
→ Обучение (Python): симулятор → 30 признаков → LightGBM → ONNX.
→ В приложении: история → признаки → прогноз, на устройстве.
→ Точность: MAPE 2,5% (~1,8 кг), точнее базовых методов.

**7. Синтетические данные**
→ Реальных данных нет, соревновательные базы — без истории.
→ Симулятор атлета по силовым нормативам: рост→плато, усталость, травмы, детренинг.
→ 500 атлетов, 78 тыс. пар. Разбиение по атлетам — учит закономерности, не людей.

**8. Два сценария**
→ Линии: рабочий вес / расчётный 1ПМ (Эпли) / прогноз модели.
→ Сценарий 1: прогресс → прогноз растёт.
→ Сценарий 2: спад (травма) → прогноз вниз. Не подталкивает к перегрузке.

**9. Выводы**
→ Все задачи выполнены: приложение + модуль через ONNX.
→ Точность ~2,5%. **Цель достигнута.**
→ Перспективы: упражнения, iOS, реальные данные.

**10. Спасибо**
→ Доклад окончен. Благодарю за внимание, готов к вопросам.
