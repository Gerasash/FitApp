## Слайд 2. Актуальность и постановка проблемы

**Актуальность:**
- Силовые тренировки требуют системного учёта: вес, повторения, подходы.
- Существующие приложения — только мобильные, с подпиской и обязательным интернетом.
- Ни одно популярное решение не прогнозирует развитие результата.

**Проблема:** нет доступного кроссплатформенного приложения, которое ведёт учёт офлайн и прогнозирует силовые показатели.

## Слайд 3. Цель и задачи

**Цель:** повышение эффективности тренировочного процесса спортсменов за счёт автоматизации учёта нагрузок и интеллектуальных рекомендаций по планированию.

**Задачи:**
1. Проанализировать предметную область и аналоги, сформулировать требования.
2. Спроектировать и реализовать кроссплатформенное приложение на .NET MAUI (MVVM, SQLite).
3. Разработать и обучить ML-модель прогноза 1ПМ, интегрировать через ONNX.
4. Протестировать приложение и оценить точность модели.

## Слайд 7. Обучающие данные — синтетический датасет

*(симулятор атлетов — самое уязвимое место, выносим отдельно)*

**Почему синтетика:**
- Открытых наборов с историей тренировок пользователя не существует.
- Соревновательные базы (OpenPowerlifting) — только разовые результаты, без истории между стартами.
- → единственный способ получить размеченные данные нужной структуры.

**Параметрический генератор виртуального атлета** (откалиброван по strengthlevel.com / Symmetric Strength):
- логистический рост 1ПМ к генетическому потолку (новичок — быстро, опытный — плато)
- подбор рабочего веса по обратной формуле Эпли с учётом RPE
- усталость/восстановление по мышечным группам, травмы, детренинг, мышечная память

**Масштаб:** 500 атлетов · 763 322 подхода · 78 384 обучающих пары · разбиение по атлетам (без утечек).
