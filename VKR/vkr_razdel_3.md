# 3. РАЗРАБОТКА КЛИЕНТСКОГО ПРИЛОЖЕНИЯ

## 3.1 Структура навигации и основные экраны

Навигация в приложении построена на основе Shell с нижней панелью вкладок. Приложение содержит семь основных экранов:

*[Рисунок 3.1 — Схема навигации клиентского приложения FitApp — файл: `diagrams/diagram_3_1_navigation.drawio`]*

1. **WorkoutListPage** — список тренировок пользователя с датами и мышечными группами; точка входа для создания новых тренировок.
2. **WorkoutPage** — страница активной тренировки: добавление упражнений и подходов, RPE, отображение ML-подсказки («Через 28 дн.: ~152 кг») и рекомендации планировщика.
3. **ExercisePage** — каталог упражнений с фильтрацией по мышечным группам.
4. **ProgressPage** — страница статистики: тоннаж за период, тепловая карта активности, рекорды, график динамики 1ПМ.
5. **TemplatesPage / TemplateEditorPage** — создание и редактирование шаблонов тренировок.
6. **SettingsPage** — профиль пользователя (вес тела, возраст, пол, целевой RPE).
7. **AccountPage** — вход/регистрация и управление синхронизацией.

## 3.2 Модуль ведения тренировок

Основной пользовательский сценарий реализован на `WorkoutPage`. При загрузке страницы `WorkoutViewModel.InitAsync()` запрашивает из локальной БД список упражнений тренировки, а для каждого из них асинхронно вызываются `OnnxPredictionService.PredictAsync()` и `WorkoutPlannerService.GetRecommendationAsync()`. Результаты отображаются в виде вспомогательного текста рядом с упражнением.

Добавление подхода выполняется командой `AddSetCommand`, которая создаёт запись `ExerciseSet` в локальной БД с `UpdatedAt = DateTime.UtcNow`. Такой подход обеспечивает немедленное сохранение без явного нажатия «Сохранить» и исключает потерю данных при закрытии приложения.

Команда «Применить план» (`ApplyPlanCommand`) создаёт в базе рекомендованное число подходов с рассчитанным весом и повторениями, оставляя поле RPE пустым для заполнения пользователем по факту выполнения.

## 3.3 Модуль статистики и прогресса

Страница прогресса предоставляет несколько агрегированных представлений данных, реализованных через SQL-запросы в `WorkoutDataBase`:

- **Тоннаж** (суммарный объём нагрузки за период, кг × повторения) — отображается по неделям в виде столбчатой диаграммы;
- **Личные рекорды** — максимальный 1ПМ по каждому упражнению за всё время;
- **Тепловая карта активности** — матрица «неделя × день недели», закрашенная пропорционально числу выполненных подходов (аналог GitHub contributions graph);
- **График динамики 1ПМ** — ломаная по тренировкам с линией тренда (линейная регрессия по последним 5 точкам).

Для отрисовки диаграмм используется библиотека **LiveChartsCore.SkiaSharpView.Maui 2.0**, поддерживающая аппаратное ускорение на Android и Windows.

## 3.4 Каталог упражнений и шаблоны

Каталог упражнений содержит более 50 упражнений, предварительно загруженных при первом запуске приложения. Каждое упражнение характеризуется типом оборудования, основными и вспомогательными мышечными группами, флагом `IsCompound`. Пользователь может добавлять собственные упражнения.

Шаблоны тренировок позволяют сохранить набор упражнений с целевым числом подходов и повторений и использовать их для быстрого создания новой тренировки. Редактирование шаблона выполняется на странице `TemplateEditorPage` с поддержкой drag-and-drop переупорядочивания.
